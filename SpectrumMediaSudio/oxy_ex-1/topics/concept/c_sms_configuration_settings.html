Settings

The Settings panel is used to declare all servers, like Spectrum X, that are connected to the Spectrum Media Studio and to configure the different functions that are used in the different desktops.

Warning: Before starting the configuration, make sure that all necessary licenses are present. To check the licenses installed on your Spectrum Media Studio, refer to License tab.

Click on the button "plus" to add any server / functionClick on the button "bin" to delete any selected server / function

Servers

To add a server, click + icon:

  • Server Address: management IP address of the server.
  • Alias: name of the server that is used in the configuration of the different functions
  • Get: to get the list of players associated to the server
  • Players: from the drop down menu, select the Player

To remove a server, select a server from the list and then click the bin icon.

Click Save to apply any change.

Player

Used to define the Players and the Recorders.

  • Server: select a server, already declared in the servers tab.
  • Player: select a player already defined in the selected server.
  • Name: name to be displayed into the application.
  • Default Playlist: NOT USED.
  • Streaming port: If the Spectrum server is running the correct firmware and is licensed to use the confidence monitor, then you can add the URL for the stream that matches this player.
    • example: <IP of spectrum director>:7087/janus:<monitor id> eghttps://10.10.10.10:7087/janus:1
  • AsRun: Selecting this option causes the playlist module to create an AsRun logs in <install folder>\AsRun\<date>. A new file is created each time a playlist is activated.
  • NDF: When using 29.97 or 59.94 players Spectrum Media Studio assumes all clips are Drop Frame. If the operation requires Non Drop Frame select this option.
  • Write In.Out in file: When selected the Mark In and Mark out in the player writes directly to the file loaded. If unchecked, then the user will be presented with a button in the player to write the Mark In and Out to the file.
Warning: When sub clipping, makes sure this option is unchecked.
  • Keep Played Item: Keeps “done” items when using the playlist panel.
  • Use Notification: Not implemented in this release.
  • Title column visible: uncheck this box to hide the title column in the player panel.
  • Eject on close: If box checked, when closing a Player / Recorder panel, the clips are ejected.
  • Service x: STL file management, used to identify the subtitle service in a player. The number of service Id depends on the subtitle services configured on the System Manager.
  • Page x: Txt page as defined in system manager for STL insertion service.
  • Color: Choose a color that appears at the top bar of a Player/Recorder panel, to identify it.
  • Stocks: select the list of stocks available for this player.
  • Main Stock: select the stock to be used to load the video clips.
  • Associated Mixer: From version 1.5 it is possible to switch to/from a live event in the playlist module. A mixer must be licensed and have been created and the name of that mixer entered here. For configuration refer to Playlist Module XXXXXXXXXXXXXXXXX.
  • Transfer Stock: From version 1.5 it is possible to specify an external location for the playlist module to fetch “missing” clips from.
  • URL Icone: Users can customize the playlist panel with a channel icon for example. Enter in this box the path to the .png required. Eg. C:\SMS\SMS-1.5.1.7\Config\Icons\channel.png
Warning:

Icons should be stored in the <SMS install path>\Config\Icons\ folder.

Icons must be a .PNG 400x 200 pixels.

DO NOT remove the default icons black.png and harmonic.png.

Configuring LOOP Playback

The loop playback and extract option require the installation of a new service and addition of some entries into the FiT service stock.xml.

  1. Installation of ServiceExtract
    1. Unzip the supplied ServiceExtract.zip into the same folder as your SMS and FIT services
    2. Inside the ServiceExtract folder locate “_serviceinstall.bat” file. Right click on the file and select “Run as Administrator”. Once the service has installed, press any key to close the CMD window.
    3. Confirm the service is running under windows services
  2. Configuration of ServiceExtract
    1. The extract service has only one configuration file and this should be left at defaults :

      C:\SMS\ServiceExtract\App_Data\Configuration\ApplicationSettings.json

  3. Configuration of sms.exe.config

    Locate the “sms.exe.config” file your main installation folder. In the section <appSettings> add the following keys

    <add key="UrlExtractService" value="http://127.0.0.1:9540" /> (IP and port where the extract service is running)

    <add key="PathExtract" value="C:\Temp\AsRun" /> (for future use)

    <add key="NbValidClipLoop" value="10" /> (number of segments to load on the timeline)*

    * This number will be dependent on the duration of the loop and segment size as is designed to prevent orphaned or locked clips being loaded inadvertently. For example, Spectrum loop service is running a 24 loop with 1hr segments, this would create 25 clips, 24 + 1 that currently in record. So if the user need to see the last 24 hours then they can enter 25 here. This currently affects ALL loop panels and will be improved in a later version.

  4. Configure FiT Service – Stock.XML

    An entry for each loop record needs to be created in stock.xml. For Example

    </Stock>

    <Stock Name="LOOP-4" Host="10.102.210.200" Port="21" User="user" Password="user" InitialDir="/">

    <Protocole>FTP</Protocole>

    <Mode>Passive</Mode>

    <ClipPath>/fs0/loopRecord/clips/LOOP-4</ClipPath>

    <SubtitlePath>/fs0/clip.dir/subtitle.dir</SubtitlePath>

    <GraphicsPath>/fs0/gfx.dir</GraphicsPath>

    <MaxOutGoingTransfert>1</MaxOutGoingTransfert>

    <MaxIncomingTransfert>1</MaxIncomingTransfert>

    </Stock>

    The clip path must point to the folder created by the loop service.

    By default, all extracted clips are stored in /fs0/loopRecord/clips/LOOP-4/Extract

    If the user wants to be able to extract to a different location, then this must also be configured in stock.xml.

  5. Configuring Player Settings
    • Server - Select Spectrum server, already configured in Servers Settings.
    • Player – Select player from list retrieved from Servers Settings.
    • Name – Enter a friendly name.
    • Default Playlist – For LOOP players this is set to the default folder the extract service will create sub clips, in this case - /fs0/loopRecord/clips/LOOP-4/Extract
    • Streaming port – Confidence monitor settings eg, "https://10.102.210.200:7087/janus:0" (assumes supported Spectrum hardware and Spectrum license
    • Main Stock – Select the LOOP folder configured in “Stock.xml”
    • Transfer Stock – Select a stock option here if the user wants to select a different folder to create the extracted clips.
    Important: Make sure "Is Loop" is selected:

Recorder

Used to select the Recorders and associated settings.

  • Server: select a server, already declared in the servers tab.
  • Recorder: select a recorder already defined in the selected server.
  • Name: name to be displayed into the application.
  • Default Path: NOT USED.
  • Streaming port: If the Spectrum server is running the correct firmware and is licensed to use the confidence monitor, then you can add the URL for the stream that matches this recorder.
    • example: <IP of spectrum director>:7087/janus:<monitor id> eghttps://10.10.10.10:7087/janus:1
  • Sources Group: If the router service is configured, select the required group to display in the recorder.
  • Router output: Logical output port of the router that is connected to the recorder.
  • Color: Choose a color that appears at the top bar of a Player/Recorder panel, to identify it.
  • Name for Rec: Name used instead of the player's name when using the template option for the recorders.
  • Use RefVITC: When selected, the Program records uses the RefVITC on the Spectrum as the time reference. Requires that the reference fed to the spectrum contains valid VITC. If not then house time is used.

  • NDF: When using 29.97 or 59.94 players Spectrum Media Studio assumes all clips are Drop Frame. If the operation requires Non Drop Frame select this option.

Multiplayer

Used to define Multiplayer module.

  • Name: name of the Multiplayer displayed at the top of the module.
  • Player List: select the players into the Multiplayer.
  • Pause on next: If selected, when playing a clip and clicking ‘next’, it pauses on the first frame of the next clip. If not selected, the next clip is played automatically.
  • Shortcut Player 1 to 4: select the active player.
  • Shortcut Play/Pause: for the button "Play/Pause".
  • Shortcut clear: for the button "Clear".
  • Shortcut Load/Eject: for the button "Load/Eject".
  • Shortcut Som: for the button "Som".
  • Shortcut Next: for the button "Next".
  • Columns Visibility: for all players, select which columns to be displayed in the playlist area.

Gang Record

Attention: This function is for testing only in this release.

This option allows users to gang together a group of previously configured recorders.

To do so, add a Multi-recorder instance and from the drop down menu select the recorders to add.

Users can different groups of recorders.

Mixers

Used to define a Mixer panel.

  • Server: select the Server (with Oxtel port) to be used.
  • Oxtel Port: TCP port to use the Oxtel protocol for advanced functions.
  • Desc: name of the Mixer into the application.
  • Audio: if selected, displays the audio profile for each source.
  • Profile: selection between: SC1 - SC2 - EC1 - EC2 - EC3 - UHD1. This should match the setting in your Spectrum channel configuration.
  • Dsp Preview: this option is used in conjunction with independent branding on the secondary output of the Spectrum to provide a preview of the off-air source on the mixer. This function is now available directly from a custom template.
  • Keyer Preview: Layer for the template to be called if Dsp Preview is activated.
  • Transition Mode: NOT USED.
  • Duration: NOT USED.
  • Associated Mixer: Add a slave mixer to a primary mixer. A mixer must be created for each main and backup channel.
  • Input 1 to 8: select the inputs you require to be displayed in the Mixer panel and give then a display name.
  • SWF: template to be called for this source if Dsp Preview is activated.
  • Out: when using router control, users can set the router port that the inputs to the Spectrum are connected to.
Note: According to the Profile selected, the number of inputs may vary.

Keyers

Used to setup the Keyers bar.

  • Server: select the Spectrum Server from drop down list.
  • Oxtel Port: Enter the oxtel port of the channel this keyer will connect to (Consult Spectrum documentation to ascertain correct port number).
  • Desc: Enter a user-friendly description for the keyer bar.
  • Keyer Preview: NOT USED.
  • Size: Select whether 4 or 8 buttons are displayed.
  • Vertical Orientation: Future release.
  • Confirm on Clear: Prompts user to confirm operation of the “Clear All” button.
  • Confirm OnAir Change: Prompts the user if they attempt to drag and drop a different template onto a layer that is already on air.
  • Erase on Clear: Unloads all templates from the layers when the “Clear All” button is pressed.
  • Unload OnAir: Prompts the user that they are about to unload a layer if it is still on air.
  • Ass. Keyer: Add a slave keyer to a primary keyer. A keyer must be created for each main and backup channel.
  • Transition Fade : From version 1.6.3.0 it is possible to set the keyer bar default transition to fade and set the transition time in frames.
  • Template 1 to 8: NOT USED in the advanced template option.
  • ... : NOT USED.

Macro Editor

Used to create new macros and new panels of macros.

Attention: First, it is necessary to create macros before to inserting them into macro panels that can be displayed on the desktop. General

Used to setup the default parameters for the Recorders.

Server Timecode Tuning:

  • Default Duration: default duration for a live/crash record.
  • Cue Preroll: used to define how long before a program record, the system will cue the recorder.
  • Default Start: not currently implemented.
  • Freeze: used for Playlist and Multiplayer panels
    • If set to 00:00:00:00 = Clips in a manual playlist will hold indefinitely on the last frame of the clip.
    • If set to 00:00:05:00 = Clips in a manual playlist will hold for 5 seconds on the last frame of the playing clip, then the next clip will be cued and hold on its first frame.
  • Template Name: template of the file name for a Crash record.
  • Use template: to select if using a template for the Crash record.
  • Recurrent Template: This template will be applied to any recurring recording setup in the recorder panel- “Is Weekly” and “Is Daily”.
  • Use SetTitle: used in conjunction with the metadata feature. Users can add a "Title" in the recorder panel. This title is added to the metadata field in the recorded file.
  • Update Sirius: used in conjunction with Sirius application. When selected the title field is updated in the Sirius application.
  • Truncate Playlist: For future use.
  • Insert Playlist Manual: When adding clips to a playlist this option sets the default play mode of the clips, when checked clips is set to Manual mode.
  • Double Click Playlist Mode: When selected this sets the playlist into shot box mode. Double clicking on any clip in the playlist ejects the current clip and cues the selected clip.
  • Count Record from zero : When checked the record panels will show the record time counting from zero upwards. When un-checked the record panel will show the actual timecode being recorded by the recorder.

Use template

If "Use template" is activated, Template Name allows:

_dd_MM_yyyy_hh_mm_ss

  • dd = day
  • MM = month
  • yyyy = year
  • HH = hour
  • mm = minute
  • ss = second

If nothing is set in Clip in the Recorder module, the name will be: "NameRecoder_29_06_2020_15_39_12.mxf"

If a name is set in Clip in the Recorder module, the name will be: "NameRecoder_29_06_2020_15_39_12_ClipName.mxf"

Then, any combination of _dd_MM_yyyy_hh_mm_ss is possible, for instance:

  • American date = _MM_dd_yyyy_HH_mm_ss
  • Only the time = _HH_mm_ss
  • etc.
